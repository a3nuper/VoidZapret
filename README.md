# VoidZapret

**VoidZapret** — графическая оболочка (GUI) для [`zapret`](https://github.com/Flowseal/zapret-discord-youtube) на Windows. Запускает обход DPI-блокировок в пару кликов: не нужно вручную возиться с `.bat`-файлами, выбирать стратегии и держать открытыми чёрные окна командной строки.

Под капотом используется `winws.exe` + драйвер `WinDivert` (тот же движок, что и в сборке Flowseal), а сверху — современный тёмный интерфейс **«Midnight glass»** на **WebView2** (HTML/CSS/JS) с настоящим стеклом, свечением и плавными анимациями. Python-ядро (оркестратор обхода, тестер, апдейтер) общается с фронтом через мост `pywebview`. Есть автоподбор рабочего метода, самопроверка соединения, watchdog, авто-обновление и установщик с ярлыком на рабочий стол.

> Назначение — восстановление доступа к ресурсам, которые ограничивает DPI провайдера (YouTube, Discord, игровые сервисы и т.д.).

---

## Возможности

- **Обход в один тумблер** — никаких ручных запусков `.bat`, окон cmd и т.п.
- **Умный обход (оркестратор)** — единый управляющий слой поверх `winws`: сам подбирает рабочую стратегию, поднимает её и в фоне следит за связью.
- **Всё в одном тумблере** — никакого выбора профилей: оркестратор всегда поднимает `combined` (сайты + Discord + DBD одновременно), а `general*.bat` держит как запасные стратегии.
- **Watchdog** — пока обход активен, оркестратор периодически перепроверяет связность; если `winws` упал или DPI «протух» (связь пропала) — **автоматически переключается** на другую рабочую стратегию без участия пользователя.
- **Автоподбор лучшего метода** — перебор стратегий с реальными сетевыми проверками (TLS-рукопожатие к обычно блокируемым хостам) и выбором рабочего варианта.
- **Тёмный интерфейс «Midnight glass»** — современный дизайн с indigo→violet градиентом, светящимся индикатором статуса и стеклянными карточками.
- **Самопроверка** — после включения обход автоматически проверяется; если стратегия не пробивает — перебираются запасные.
- **Скрытые окна** — консольные окна `winws` прячутся, на экране ничего не мелькает.
- **Сворачивание в трей** — закрытие окна прячет приложение в системный трей; обход и watchdog продолжают работать в фоне, окно возвращается из меню трея (отключается в настройках).
- **Запоминание настроек** — выбранные стратегии и состояние обходов восстанавливаются при следующем запуске (и переживают переустановку).
- **Подмена DNS** — отдельная вкладка: принудительный Google DNS + OpenDNS на активные адаптеры (обход блокировок уровня DNS) или выбор провайдера (Cloudflare / Google / Dns.SB) с best-effort DoH, и сброс на DHCP.
- **Cloudflare WARP** — управление через `warp-cli` на отдельной вкладке.
- **Обновление `zapret`** — ручное и авто-обновление встроенных стратегий из релизов Flowseal.
- **Установщик** — Inno Setup с ярлыками на рабочем столе и в меню «Пуск», правами администратора и чистой переустановкой.
- **Свой движок VoidEngine** (в работе) — собственный, независимый от Flowseal обход на WinDivert: перехват TLS ClientHello и десинк (split/fake/seqovl) с авто-калибровкой и таргетингом по хостам. Пробуется **первым** при нажатии «Запустить»; combined/general — фолбэк.
- **Авто-обновление приложения** — проверка новых версий VoidZapret через GitHub Releases (тумблер «Автообновление»).

## Как это работает

### Движок
Обход выполняет `winws.exe` (из комплекта Flowseal) с драйвером `WinDivert`. Он перехватывает исходящие TLS ClientHello и QUIC и применяет техники десинхронизации DPI (`fake`, `multisplit`, поддельные QUIC/STUN/Discord-пакеты, split-seqovl и т.д.). VoidZapret сам формирует и запускает нужный `.bat`, отслеживает процесс и показывает статус.

### Оркестратор (умный обход)
Вместо отдельных тумблеров обходом управляет один слой — **оркестратор** ([`core/orchestrator.py`](core/orchestrator.py)):

- На вход подаётся список стратегий-кандидатов (`combined.bat` + `general*.bat`) и список целевых хостов.
- При запуске оркестратор перебирает кандидаты (предпочтительная — первой), поднимает `winws`, ждёт инициализацию и проверяет реальную связь до целей. Первая прошедшая стратегия становится активной (**самопроверка + авто-перебор**).
- **Watchdog** в фоне периодически перепроверяет связность; при падении `winws` или просадке связи на лету переключается на другую рабочую стратегию (плохую отодвигает в конец очереди).

### Один активный обход (важно)
`winws` **не запускает второй экземпляр с пересекающимся фильтром** — он сразу выходит с сообщением `A copy of winws is already running with the same filter`. Поскольку все стратегии используют порт 443, нельзя держать general + Discord + DBD как **отдельные** процессы. Поэтому:

- активен **один** `winws` за раз;
- основная стратегия — **`combined.bat`**: это **один** `winws` со множеством групп фильтров (`--new`) сразу: Discord-голос/STUN, домены Discord, домены Epic/EOS, игровые порты, общий веб. Именно для этого `winws` и спроектирован. `general*.bat` подключаются как запасные, если `combined` не пробивает.

### Отслеживание процесса и скрытие окон
- `winws` запускается из `.bat` **напрямую** (без `start`), а сам `cmd` поднимается с флагом `CREATE_NO_WINDOW`. Так `winws` наследует скрытую консоль — окна нет, а процесс жив как дочерний.
- Каждый обход отслеживает **свои** PID `winws` (через diff `tasklist`), поэтому статус и остановка точны.
- На старте приложение убивает «осиротевшие» `winws` от прошлых аварийных запусков (иначе зомби-процесс держит драйвер `WinDivert`, и новые обходы не стартуют).

### Самопроверка
После запуска стратегии выполняется TLS-проверка набора обычно блокируемых хостов (для веб — цели из `targets.txt`, для Discord/DBD/combined — соответствующие домены). Если успешных соединений достаточно — обход считается рабочим; иначе перебираются запасные стратегии.

## Метод обхода

Основной метод — **наш движок VoidEngine** (см. [`core/engine/`](core/engine)): независимый от Flowseal перехват TLS ClientHello на WinDivert с десинком (`fakedisorder / disorder / fakesplit / fake / split / seqovl`), `ip.ident=0` (фикс Google/YouTube-видео), таргетингом по встроенному списку хостов и авто-калибровкой (берётся техника, прошедшая самопроверку). Пока активен — глушит QUIC, чтобы Instagram/игры/видео шли по TCP, который движок пробивает. Главное преимущество — **низкий пинг** (десинк только нужных SNI).

Запасной вариант (на случай, если движок где-то не пробьёт) — `combined (ALT10/ALT11).bat` (один `winws` со всеми группами фильтров). Старые `general*` удалены как устаревшие; `combined*` не входит в релиз Flowseal и сохраняется при обновлении.

> **Формат `.bat` критичен:** файлы должны быть в кодировке с **CRLF**-переносами и **ASCII-комментариями**. `cmd.exe` некорректно разбирает `.bat` с LF-переносами или многобайтными символами в комментариях после `chcp 65001`.

> **Формат `.bat` критичен:** файлы должны быть в кодировке с **CRLF**-переносами и **ASCII-комментариями**. `cmd.exe` некорректно разбирает `.bat` с LF-переносами или многобайтными символами в комментариях после `chcp 65001` (ломается `call service.bat`, обход не инициализируется).

## Установка (готовая)

Скачай последний установщик со страницы **[Releases](https://github.com/a3nuper/VoidZapret/releases/latest)** (`VoidZapret-Setup-*.exe`) и запусти. Он поставит приложение, при необходимости тихо доустановит WebView2 и создаст ярлыки. Приложение само проверяет новые версии при запуске (авто-обновление через GitHub Releases).

---

## Запуск из исходников

```powershell
py -m pip install -r requirements.txt
py main.py
```

- Python вызывается через лаунчер **`py`** (не `python` — в Windows это часто заглушка из Microsoft Store).
- При старте приложение **повышает права до администратора** (UAC), т.к. `winws` использует драйвер `WinDivert`.
- Интерфейс рендерит **WebView2** (Edge Chromium) — рантайм предустановлен в Windows 10/11. Зависимости: `pywebview` + `pythonnet`.

---

## Сборка `.exe`

Сборка делается через **PyInstaller в режиме onedir** (папка с `exe` + `_internal`):

```powershell
py -m PyInstaller --noconfirm VoidZapret.spec
```

Результат: `dist/VoidZapret/VoidZapret.exe` (+ папка `_internal` со встроенным `zapret`, `webui` и DLL WebView2/pythonnet).

Особенности сборки:
- **WebView2 + pythonnet** — spec собирает их через `collect_all('webview'/'pythonnet'/'clr_loader')`, фронт `webui/` кладётся в datas. WebView2 Runtime должен быть в системе (есть в Win10/11; иначе — тихий bootstrapper в установщике).
- **onedir, а не onefile** — у onefile архив дописывается «хвостом» в `.exe`, и Windows Defender нередко вырезает его у неподписанных сборок (ошибка `Could not load PyInstaller's embedded PKG archive`). onedir этим не страдает.
- **UPX отключён** — провоцирует ложные срабатывания антивируса.
- В манифест зашит `requireAdministrator` (`uac_admin`).
- Иконка берётся из `icon.ico` (генерируется из `logo.png` через `py make_icon.py`).

## Установщик

Скрипт `installer/VoidZapret.iss` (компилятор — Inno Setup):

```powershell
# Inno Setup ставится через: winget install -e --id JRSoftware.InnoSetup
ISCC.exe installer\VoidZapret.iss
```

Результат: `dist/installer/VoidZapret-Setup-<версия>.exe`.

Установщик:
- ставит программу в `Program Files\VoidZapret` с правами администратора;
- создаёт ярлык на **рабочем столе пользователя** и в меню «Пуск»;
- при переустановке **чисто** удаляет старое содержимое, но **сохраняет конфиг** (запомненные стратегии и настройки);
- русский и английский языки мастера;
- **тихо доустанавливает WebView2 Runtime** (`MicrosoftEdgeWebview2Setup.exe`), если его нет в системе;
- предлагает запустить приложение по завершении.

## Обновление zapret

Источник релизов: `https://github.com/Flowseal/zapret-discord-youtube`.

- Версия и архив берутся через `raw.githubusercontent.com` + `codeload` (без лимитов `api.github.com`).
- Архив скачивается, распаковывается в постоянную папку `zapret` рядом с `exe`, кастомные `combined*` стратегии сохраняются, а `general` триммится до ALT10/11/12.
- Доступно как ручной кнопкой в «Настройках», так и авто-обновлением при запуске (с уведомлением и предложением заново подобрать метод).

## Требования

- Windows 10/11 (x64);
- права администратора (для `winws` / `WinDivert`);
- для запуска из исходников — Python 3.10+ (рекомендуется 3.12+), зависимости из `requirements.txt`.

## Примечания

- Приложение и установщик **не подписаны** цифровой подписью, поэтому Windows SmartScreen или антивирус могут разово предупредить при первом запуске — это ожидаемо для неподписанного ПО.
- Эффективность конкретной стратегии зависит от провайдера и его DPI: если автоподбор/самопроверка не находит рабочий вариант, попробуйте другие стратегии или обновите `zapret`.
- VoidZapret — обёртка; ядро обхода и стратегии основаны на проекте [Flowseal/zapret-discord-youtube](https://github.com/Flowseal/zapret-discord-youtube).
